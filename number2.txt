
Завдання №2
(2 години) Додаємо можливість видаляти рядки з таблички.
Як це буде виглядати:
У таблиці з'явиться стовпчик "Дії" і в кожному рядку в цьому стовпчику — 
кнопка червоного кольору з текстом "Видалити".
По кліку кнопку — рядок пропадає з таблиці.
Реалізувати це теж максимально просто: потрібно натиснути кнопку 
видалення послати DELETE запит на apiUrl/<id>. Наприклад для першого 
користувача, запит йтиме на урл схожий на https://somesite.com/users/1. 
ID потрібно отримувати з івента натискання на кнопку. Як це можна зробити:

Перший варіант: 
При рендері таблички, можна класти id в data-id і по кліку брати його з атрибутів кнопки.
<button data-id="45">delete</button>
function deleteUser(event) {
  // отримує з event кнопку і серед її атрибутів id, потім відправляє запит
}
// … тут цикл, який проходиться по юзерам і створює кожному кнопку при рендері
button.onclick = deleteUser;

Другий варіант:
Коли ви в циклі рендерите кожний рядок у табличці, навішувати слухач на кожну конкретну кнопку
// … тут цикл який проходить по користувачах і рендерить кнопки для кожного
button.onclick = function () { deleteUser(userId) } // userId береться з циклу
У кожному варіанті є свої плюси та мінуси, подумайте які і виберіть той, який вважаєте найоптимальнішим. Не забудьте на рев’ю обговорити ваш вибір.
Для цього завдання та для наступного:
Після того, як піде запит, вам потрібно отримати оновлений список даних з бекенду і відрендерити його заново (щоб підтягнулися зміни, які ви зробили).

Завдання №3
(4 години) Додаємо можливість додавати нові записи до таблички.
Як це виглядає:
Вгорі таблиці, поряд з інпутом для пошуку, додаємо кнопку "Додати", 
при натисканні на яку додається рядок вгорі таблички, але замість 
тексту в колонках інпути. Після натискання на enter на кожному з 
полів необхідно спробувати додати користувача. Але перед цим, звичайно, 
перевірити, чи всі поля заповнив користувач. Якщо лишилися порожні, 
потрібно підсвітити їх червоною рамкою.
Якщо все заповнено правильно, відправляєте POST запит на apiUrl, з 
усіма даними про користувача у форматі JSON і назвами полів такими 
ж, як у конфігурації.
